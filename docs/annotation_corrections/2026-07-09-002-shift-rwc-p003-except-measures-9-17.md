# 2026-07-09 002 Shift RWC P003 Except Measures 9 To 17

Status: validated

## Scope

- Annotation type: aligned MIDI vocal track
- RWCID: `RWC_P003`
- Input files:
  - `00_annotations_original/MIDI_aligned/metadata_mapping_vocals.csv`
  - `00_annotations_original/MIDI_aligned/RWC-P/RWC_P003.mid`
- Output files:
  - `01_annotations_preprocessed/MIDI_aligned/RWC-P/RWC_P003.mid`
- Script:
  - `scripts/annotation_corrections/002_shift_rwc_p003_except_measures_9_17.py`

## Problem

`RWC_P003` has a non-empty comment in `metadata_mapping_vocals.csv`, so it was intentionally excluded from the first global blank-comment correction.
The required correction is partial: keep measures 9 through 17 at the original pitch, and shift all other vocal notes down by 12 semitones.

## Evidence

- Mapping row: `RWC_P003;4;MELODY;12;from 17 to 35 seconds one octave lower`.
- The MIDI track check shows `RWC_P003` track 4 is named `MELODY`.
- The MIDI file uses 4/4 time with 480 ticks per quarter note.
- Using 1-based measure numbering, measures 9 through 17 correspond to ticks `[15360, 32640)`.

## Decision

- Add `RWC_P003` as a comment-specific vocal shift target.
- Shift track 4 down by 12 semitones outside measures 9 through 17.
- Preserve note-on events that start inside measures 9 through 17.
- Shift note-off events according to their matching note-on decision so MIDI note durations remain valid across boundaries.
- Leave timing, velocities, metadata, track order, and chunk lengths unchanged.

## Implementation

```powershell
python scripts\annotation_corrections\apply_all.py
```

The correction module `002_shift_rwc_p003_except_measures_9_17.py` defines the P003-specific rule. Shared MIDI helpers support per-target preserved measure ranges and convert those ranges to MIDI ticks from the file division. The module is invoked through `scripts/annotation_corrections/apply_all.py`.
